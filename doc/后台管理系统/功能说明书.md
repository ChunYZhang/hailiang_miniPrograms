## 一、系统模块总览

后台管理系统分为以下模块：

| 模块 | 说明 |
|------|------|
| 首页/控制台 | 数据概览统计、图表 |
| 娃娃管理 | 娃娃的增删改查、上下架、热门标记 |
| 配饰管理 | 配饰的增删改查、上下架、热门标记 |
| 系列管理 | 娃娃系列维护 |
| 分类管理 | 配饰分类维护 |
| 搭配方案 | 系统预设搭配方案管理 |
| 询价订单 | 用户提交的询价单管理、跟进 |
| 用户管理 | 小程序注册用户审核、管理 |
| 库存管理 | 库存记录、进出台账 |
| 报表统计 | 数据图表、导出 |
| 公司信息 | 公司联系方式、资质证书 |
| 轮播图管理 | 首页轮播图管理 |
| 邮件配置 | SMTP邮件通知配置 |
| 员工管理 | 后台账号管理 |
| 操作日志 | 系统操作记录 |

## 二、娃娃管理

### 2.1 功能说明

管理娃娃商品，包括名称、价格、起购数量、箱子容量、图片等。

### 2.2 页面文件

- 前端页面：`frontend/src/pages/Products/DollsPage.tsx`

### 2.3 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/admin/doll/list | 列表（支持keyword、status、series筛选，分页） |
| POST | /api/admin/doll/create | 创建 |
| POST | /api/admin/doll/update | 更新 |
| POST | /api/admin/doll/delete | 删除 |
| POST | /api/admin/doll/toggle-status | 切换上下架 |
| POST | /api/admin/doll/toggle-hot | 切换热门标记 |

### 2.4 娃娃表结构（doll）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键UUID |
| name | VARCHAR(100) | 娃娃名称 |
| price | DECIMAL(10,2) | 单价 |
| material | VARCHAR(50) | 材质 |
| size | VARCHAR(50) | 尺寸 |
| stock | INT | 库存数量 |
| status | ENUM | active/inactive |
| series | VARCHAR(50) | 系列名称 |
| patent_no | VARCHAR(100) | 专利号 |
| description | TEXT | 描述 |
| images | JSON | 图片URL数组 |
| views | INT | 浏览量 |
| inquiries | INT | 询价数 |
| low_stock_threshold | INT | 低库存阈值 |
| min_quantity | INT | 起购数量（箱） |
| small_box_capacity | INT | 小箱可装数量 |
| medium_box_capacity | INT | 中箱可装数量 |
| large_box_capacity | INT | 大箱可装数量 |
| default_accessory | VARCHAR(100) | 默认配饰名称 |
| selected_accessories | JSON | 已选配饰数组 [{id,name,price},...] |
| is_hot | TINYINT | 是否热门 0/1 |
| created_at | DATETIME | 创建时间 |

### 2.5 创建/编辑娃娃字段

```
名称、价格、材质、尺寸、系列、专利号、描述、
起购数量（小箱容量、中箱容量、大箱容量）、
默认配饰、已选配饰（搭配方案配饰）
```

### 2.6 业务逻辑

- `is_hot=1` 的娃娃会出现在首页热门推荐
- `status=inactive` 的娃娃不在小程序端展示
- 浏览量(views)在用户访问娃娃详情页时 +1（后端自行累加）

## 七、询价订单管理

### 7.1 功能说明

查看用户提交的询价单，可更新订单状态、添加跟进记录。

### 7.2 页面文件

- 前端页面：`frontend/src/pages/Inquiries/index.tsx`

### 7.3 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/admin/inquiry/list | 列表（支持status、keyword筛选） |
| GET | /api/admin/inquiry/detail/{id} | 详情 |
| POST | /api/admin/inquiry/update-status | 更新状态+跟进备注 |
| GET | /api/admin/inquiry/follow-up/{id} | 跟进记录列表 |

### 7.4 询价订单表结构（inquiry_order）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键UUID |
| order_no | VARCHAR(30) | 订单号（如XJ20260428...） |
| user_id | VARCHAR(36) | 用户ID |
| user_name | VARCHAR(50) | 客户姓名 |
| user_phone | VARCHAR(20) | 联系电话 |
| address | VARCHAR(500) | 收货地址 |
| items | JSON | 商品明细数组 |
| total_amount | DECIMAL(12,2) | 总报价 |
| remark | TEXT | 客户备注 |
| status | ENUM | pending/contacted/quoted/closed/cancelled |
| is_pindan | TINYINT | 是否拼单 0/1 |
| admin_note | TEXT | 管理员跟进备注 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 7.5 items JSON 结构示例

```json
[
  {
    "type": "doll",
    "id": "娃娃ID",
    "name": "月光兔兔",
    "price": 36.0,
    "quantity": 500,
    "boxSize": "small",
    "defaultAccessory": "蝴蝶结",
    "accessories": [{"id":"...","name":"配饰A","price":5}]
  }
]
```

- doll类型：price = 娃娃单价 + 配饰总价；显示时 price * quantity = 小计
- quantity=1时显示娃娃单价，quantity>1时显示总价
- boxSize显示为"小箱/中箱/大箱"

### 7.6 订单状态

| 状态 | 说明 |
|------|------|
| pending | 待处理 |
| contacted | 已联系 |
| quoted | 已报价 |
| closed | 已成交 |
| cancelled | 已取消 |

### 10.5 娃娃浏览量说明

- `mobile_doll_detail` 接口每次被调用时执行 `UPDATE doll SET views=views+1`
- `mobile_doll_outfit_accessories` 接口**不增加**浏览量
- 后台图表数据从 `SELECT name, views FROM doll WHERE status='active' ORDER BY views DESC LIMIT 10` 查询

## 附录：后台前端技术栈

- 框架：React + TypeScript + Vite
- 路由：React Router v6
- UI组件：Headless UI / 自定义
- 图标：Lucide React
- 图表：Recharts（LineChart, BarChart, PieChart）
- 状态管理：Zustand（用于页面状态）
- 全局状态：React Context（用户信息）
- 样式：Tailwind CSS
- HTTP客户端：axios
- 项目路径：`frontend/src/`
