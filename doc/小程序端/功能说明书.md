## 三、娃娃列表（/pages/doll-list/index）

### 3.1 功能

展示娃娃列表，支持按系列筛选、关键词搜索。**核心功能：拼单模式**。

### 3.2 页面文件

- `miniPrograms/src/pages/doll-list/index.tsx`
- `miniPrograms/src/pages/doll-list/index.scss`

### 3.3 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/mobile/series/list | 获取系列列表（用于筛选） |
| GET | /api/mobile/doll/list | 获取娃娃列表（支持series、keyword参数） |

### 3.4 拼单模式逻辑（重要）

当用户点击"发起拼单"时，进入拼单模式：

1. **进入拼单模式**：`enterPindanMode()` 创建拼单组 `createPindanGroup()`
2. **添加商品到拼单池**：`addToPindan(item)` — 商品加入全局状态 `pindanItems`
3. **拼单池管理**：显示所有已添加的商品，支持手动修改数量、删除
4. **箱子容量控制**：
   - 根据第一个商品确定箱子尺寸（small/medium/large）
   - `boxCapacity = getBoxCapacity()` 根据尺寸返回容量（500/1000/2000）
   - `remainingCapacity = boxCapacity - pindanTotalQty`
   - `canCompletePindan = pindanItems.length > 0 && pindanItems.every(item => item.quantity >= item.minQuantity) && remainingCapacity <= 0`
5. **完成拼单条件**：
   - 所有商品都达到起购数量 `quantity >= minQuantity`
   - 箱子已满 `remainingCapacity <= 0`（可超出）
6. **超出处理**：用户可输入超过箱容量的数量，提交时备注自动注明超出
7. **取消拼单**：清空 `pindanItems`，退出拼单模式

### 3.5 拼单数据结构（PindanItem）

```typescript
{
  id: string              // 时间戳ID
  itemType: 'doll'        // 商品类型
  itemId: string          // 娃娃ID
  name: string            // 娃娃名称
  price: number           // 娃娃单价
  accessoriesPrice: number // 配饰总价
  coverImage: string      // 封面图
  quantity: number        // 数量
  accessories: Accessory[] // 已选配饰
  minQuantity: number     // 起购数量
  pindanGroupId: string   // 拼单组ID
  pindanGroupName: string // 拼单组名称
  boxSize: 'small'|'medium'|'large'  // 箱子尺寸
  smallBoxCapacity: number   // 小箱容量
  mediumBoxCapacity: number  // 中箱容量
  largeBoxCapacity: number   // 大箱容量
}
```

### 3.6 全局状态（Store）

```typescript
// miniPrograms/src/store/index.tsx
isPindanMode: boolean
pindanItems: PindanItem[]
enterPindanMode(): void
exitPindanMode(): void
addToPindan(item: PindanItem): void
updatePindanItemQuantity(itemId: string, quantity: number): void
removeFromPindan(itemId: string): void
clearPindan(): void
createPindanGroup(): { groupId: string, groupName: string }
```

## 四、娃娃详情（/pages/doll-detail/index）

### 4.1 功能

展示娃娃详细信息，支持配饰选择、加购、发起拼单。

### 4.2 页面文件

- `miniPrograms/src/pages/doll-detail/index.tsx`
- `miniPrograms/src/pages/doll-detail/index.scss`

### 4.3 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/mobile/doll/detail/{id} | 获取娃娃详情（自动 +1 浏览量） |
| GET | /api/mobile/doll/{id}/outfit-accessories | 获取娃娃已选配饰（搭配方案） |

### 4.4 配饰选择逻辑

1. 调用 `api.doll.outfitAccessories(dollId)` 获取该娃娃的预设配饰
2. 如果有预设配饰（返回非空数组），则只显示这些配饰供选择
3. 如果没有预设配饰，则显示全部配饰 `api.accessory.list()`
4. 用户选择配饰后，`savedAccessories` 保存已选配饰
5. 加入购物车时 `accessories` 参数传递已选配饰数组

### 4.5 加入购物车

```typescript
api.cart.add({
  item_type: 'doll',
  item_id: doll.id,
  accessories: savedAccessories.map(acc => ({ id: acc.id, name: acc.name, price: acc.price })),
  quantity: quantity,
  box_size: selectedBoxSize,  // 'small' | 'medium' | 'large'
})
```

### 4.6 发起拼单

```typescript
addToPindan({
  id: Date.now().toString(),
  itemType: 'doll',
  itemId: doll.id,
  name: doll.name,
  price: Number(dollPrice),
  accessoriesPrice: Number(accessoriesPrice),
  coverImage: images[0],
  quantity,
  accessories: savedAccessories,
  minQuantity: doll.minQuantity,
  pindanGroupId: groupId,
  pindanGroupName: groupName,
  boxSize: selectedBoxSize,
  smallBoxCapacity: doll.smallBoxCapacity,
  mediumBoxCapacity: doll.mediumBoxCapacity,
  largeBoxCapacity: doll.largeBoxCapacity,
})
```

### 4.7 浏览量注意

- 每次访问娃娃详情页，后端 `mobile_doll_detail` 执行 `UPDATE doll SET views=views+1`
- `mobile_doll_outfit_accessories` 不增加浏览量
- 防止 React StrictMode 重复调用问题：使用 `loadedRef` 确保只加载一次

## 十、购物车（/pages/cart/index）

### 10.1 功能

展示购物车商品，支持单选/全选、删除、编辑数量、编辑配饰、结算。

### 10.2 页面文件

- `miniPrograms/src/pages/cart/index.tsx`
- `miniPrograms/src/pages/cart/index.scss`

### 10.3 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/mobile/cart | 获取购物车列表 |
| PUT | /api/mobile/cart/{id} | 更新数量或配饰 |
| DELETE | /api/mobile/cart/{id} | 删除商品 |
| POST | /api/mobile/cart/clear | 清空购物车 |

### 10.4 购物车数据结构（CartItem）

```typescript
{
  id: string,              // 购物车记录ID
  item_type: 'doll'|'accessory'|'outfit',
  item_id: string,
  name: string,
  price: number,           // 单价（娃娃=娃娃单价+配饰总价）
  coverImage: string,
  quantity: number,
  boxSize: 'small'|'medium'|'large',
  pindan_group_id: string, // 拼单组ID（空=普通商品）
  pindan_group_name: string,
  accessories: Accessory[],  // 已选配饰
  defaultAccessories: Accessory[],  // 默认配饰（outfit类型）
  minQuantity: number,      // 起购数量（doll类型）
  smallBoxCapacity: number,
  mediumBoxCapacity: number,
  largeBoxCapacity: number,
}
```

### 10.5 数量编辑逻辑

- 点击数量区域弹出编辑弹窗
- 可手动输入数量
- **校验规则**：
  - 最低为1
  - 不能低于起购数量 `minQuantity`
  - 购物车编辑数量时不检查箱子容量（仅在拼单完成时检查）
- 输入为空时回退到原值，最小为1

### 10.6 拼单组显示

- 有 `pindan_group_id` 的商品归为拼单组，显示橙色头部标签
- 同组商品显示在一起
- 无 `pindan_group_id` 的为普通商品

### 10.7 配饰编辑

- 点击"搭配配饰"可弹出配饰选择弹窗
- 只显示与该娃娃关联的配饰（通过娃娃详情页的已选配饰或全部配饰）

### 10.8 结算流程

1. 用户勾选商品，点击"提交询价"
2. 将选中商品ID存 `wx.setStorageSync('checkout_selected_ids', JSON.stringify(selectedIds))`
3. 跳转到 `/pages/order-confirm/index`

api.cart.submit({
  user_name: selectedAddress.name,

### 13.1 功能

微信手机号授权登录为主要方式，密码登录为备选。

### 13.2 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/mobile/auth/decrypt-phone | 解密微信手机号 |
| POST | /api/mobile/auth/login | 登录 |
| POST | /api/mobile/auth/login-by-password | 密码登录 |

### 13.3 登录流程

1. 点击"微信登录"按钮
2. 通过 `wx.getPhoneNumber` 获取加密手机号
3. 调用 `api.auth.decryptPhone` 解密获取真实手机号
4. 调用 `api.auth.login` 完成登录
5. 登录成功后将用户信息存入全局状态和 Storage

## 二十、注意事项

1. **浏览量**：娃娃和配饰详情页每次加载都会 +1 浏览量，使用 `loadedRef` 防止重复
2. **拼单价格**：娃娃的 `price` 在加入购物车时为 `娃娃单价 + 配饰总价`
3. **订单提交**：询价订单的 `price` 字段存储娃娃单价（含配饰），前端显示 `price * quantity`
4. **箱子容量**：仅在拼单模式完成时检查，购物车编辑数量不检查
5. **地址格式**：提交订单时地址格式为 "省 市 区 详情"
