## 四、API 层（services/api.ts）

### 4.1 请求封装

```typescript
const BASE_URL = 'http://localhost:5000/api'

function get<T>(url: string): Promise<T>
```

所有请求自动携带 `X-Token` header（从 Storage 读取）。

### 7.5 价格计算逻辑

后端 `mobile_cart_list` 返回的 `price` 已是最终计算值，前端直接使用：

| 类型 | price含义 | 前端显示 |
|------|----------|----------|
| 拼单（pindan_group_id 有值） | 娃娃单价+配饰总价（单价） | `itemTotal = price * quantity` |
| 非拼单（按箱购买） | (娃娃+配饰)*箱子容量*箱数（总价） | `itemTotal = price`（直接使用） |

**购物车合计 `selectedTotal`**：拼单 `price * quantity`，非拼单直接用 `price`

**订单确认页合计 `totalPrice`**：拼单 `price * quantity`，非拼单直接用 `price`

## 九、注意事项

1. **图片 URL**：后端返回的图片 URL 需要拼接完整地址（如 `http://localhost:5000`）
2. **Storage Key**：
   - `userInfo` - 用户信息
   - `cartCount` - 购物车数量
   - `checkout_selected_ids` - 结算选中的购物车ID
   - `checkout_remark` - 结算备注
3. **Token 传递**：所有需要登录的请求自动从 Storage 读取 token 放入 `X-Token` header
4. **登录检测**：页面加载时检查 `userInfo`，未登录跳转到登录页
5. **Taro API**：使用 `Taro.showToast` / `Taro.navigateTo` 等，需从 `@tarojs/taro` 导入
6. **useDidShow**：页面每次 `onShow` 时执行，如娃娃列表刷新数据
